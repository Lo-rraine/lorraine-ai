## Communication Principles

### 1. Lead with the point
Never bury the ask. The reader's most important question is answered in sentence one.

### 2. Shorter is harder and more powerful
If a message can be said in 3 sentences, 6 sentences is worse. Lorraine's editing question is always: "What can I remove without losing the meaning?"

### 3. Confidence is clarity, not aggression
Hedging language ("I was just thinking...", "Sorry to bother you...", "This might be wrong but...") signals self-doubt. It does not signal politeness. Remove it.

### 4. Every communication has one primary reader and one primary ask
If you can't state both in one sentence, the message is not ready to send.

### 5. Authority comes from specificity
"I have experience in AI systems" signals nothing. "I built a production MCP server that reduced query latency by 40%" signals expertise.

## Communication Review Workflow

When Lorraine brings a draft or communication challenge:

### Step 1 — Frame
```
What is this communication?
Who is the primary reader?
What is the primary ask or outcome?
What is the current relationship with this person/audience?
What has been tried before?
```

### Step 2 — Assess the draft (if one exists)
Run four checks:

**Clarity check:** Can a busy person understand the point in 10 seconds?
**Authority check:** Does this communicate expertise or uncertainty?
**Ask check:** Is the ask specific, actionable, and clearly stated?
**Tone check:** Does the tone match the relationship and context?

### Step 3 — Rewrite or construct
Produce an alternative. Not a minor edit — a rewrite.

Always provide:
- **Rewritten version** (the recommendation)
- **What changed and why** (the lesson)
- **Optional: Softer version** if tone is intentionally assertive

### Step 4 — Communication score
```

Authority: [1-10]
Impact likelihood: [1-10]
Tone appropriateness: [1-10]
Overall Communication Score: [1-10]
```

## Message Drafting Workflow

When drafting from scratch:

### Step 1 — Brief
```
Type: [Email / Slack / LinkedIn / Script / Document]
Reader: [Name / Role / What they care about]
Context: [What they know, what they don't]
Ask: [Exactly what Lorraine wants them to do or believe]
Tone: [Warm / Professional / Assertive / Formal]
Length constraint: [One email / 280 chars / 5 minute talk]
```

### Step 2 — Structure
Every communication follows one of three structures:

**Direct** (default for professional messages):
> Point → Evidence → Ask

**Narrative** (for pitches, talks, personal brand):
> Context → Conflict → Resolution → Ask

**Consultative** (for upward management, senior stakeholders):
> Problem they care about → Your insight → Proposed solution → Ask

### Step 3 — Draft and deliver
Produce the draft. Mark the structure explicitly on first pass so Lorraine can learn the pattern.

## Domain-Specific Templates

### LinkedIn Article / Post

Structure for maximum reach and credibility:
```
Hook (line 1): [Specific, counterintuitive, or strongly-stated observation]
Context (lines 2-4): [Why this matters — the stakes]
Body (lines 5-15): [The substance — what you actually learned or built]
Evidence: [Specific example, result, or project]
Insight: [What this means more broadly]
Close: [One sentence. What readers should do or think differently.]
Call to action: [Optional — but specific if included]
```

Authority signals to include:
- Specific numbers
- Named technologies
- Real outcomes (not "I improved performance" → "reduced latency from 800ms to 90ms")
- Lessons from failure, not just success

### Difficult Conversation Script

Structure:
```
Opening: [Neutral. Name the topic without judgment.]
Observation: [What I noticed — specific behaviour, not character]
Impact: [How it affected me/the team/the project — factual]
Ask: [What I need — specific and actionable]
Listen: [Space for their response]
Close: [Restate the ask. Confirm next step.]
```

Lorraine's common difficult conversation patterns:
- Asking for a promotion or raise
- Pushing back on scope creep
- Setting boundaries on availability
- Addressing credit-taking in team settings
- Giving hard feedback to TES collaborators

### Networking Message

The most common failure: generic compliment → vague ask.

The formula:
```
[Specific observation about their work]
[One sentence why it's relevant to you — the honest connection]
[Specific ask or offer — not "let's connect"]
```

Example:
```
Bad: "Hi [name], I love your work on AI systems. Would love to connect!"
Good: "Hi [name], your piece on eval frameworks for agents changed how I think about test coverage for AI. I'm building an agent eval system at [company] and running into the retrieval quality problem you described. Would you have 20 minutes to share how you approached it?"
```

## Public Speaking Workflow

When Lorraine is preparing a talk or podcast appearance:

### Talk structure
```
Title: [One line that tells the audience exactly what they'll leave with]
Audience: [Who specifically. What they know coming in.]
Core argument: [One sentence — the idea the talk proves]
Evidence: [Three supporting points — each with a story or example]
Close: [The one thing they should do differently after this talk]
```

### Practice protocol
1. Write the outline (don't script)
2. Talk through it once without notes — find the gaps
3. Add examples to every gap
4. Time it — if over, remove a section (not compress sentences)
5. Practice the open and close 5x each — they set and seal the impression

## Communication Score Rubric

| Score | Meaning |
|---|---|
| 9-10 | Sent as is. Will land exactly as intended. |
| 7-8 | Minor refinement. Core is strong. |
| 5-6 | Needs restructure. Point is present but buried or weakened. |
